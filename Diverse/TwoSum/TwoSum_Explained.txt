Two Sum Problem

We have an array of integers and an integer target. 
Return the indices of the two numbers such that they add up to target. 

You may assume that each input would have exactly one solution, and you may not use the same element twice. 

You can return the answer in any order. 

Ex:

Input: nums = [2,7,11,15], target = 9
Output: [0,1]
Explanation: Because nums[0] + nums[1] == 9, we return [0, 1]

Input: nums = [3,3], target = 6
Output: [0,1]

Approach:

If we check all possible pairs, we need two loops -> O(n^2)

To improve the time complexity -> use a memory to keep data for something

Let's look again at the input array and target number
nums = 2, 7, 11, 15
target = 9

If we subtract the target number from the first integer, we get the number we needed to add

Ex: 9 - 2 = 7

And we want to check if 7 exists in the array, right ? 

We can use a HashMap to keep numbers (elements) as keys and index numbers as values in the input array.

Let's say we have a for where i = 0 -> nums.size() - 1

First we calculate the difference: 
dif = target - nums[i]

if we find it (m.find(dif) != m.end()) then we can save the indices 
and break the loop 

ind1 = i;
ind2 = m[dif];
break;

else (we don't explicitly write 'else' because of break instruction)

m[nums[i]] = i -> create the HashMap m[value] = index;
